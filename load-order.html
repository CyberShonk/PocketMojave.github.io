Load Order

This page covers LOOT as a helper, DML plugin sorting, and starter text files for a Pocket Mojave setup.

Use LOOT as a check, not a magic button

LOOT supports Fallout: New Vegas and can identify missing masters, warnings, dirty plugins, and order suggestions. Use it on PC when you can, then copy the result back into DML/GameNative.

  1. Install LOOT on a Windows PC or a Windows environment that can see your New Vegas folder.
  2. Make sure the same plugins are present as the DML profile you are checking.
  3. Run LOOT for Fallout: New Vegas.
  4. Read the warnings before sorting.
  5. Apply sorting only if the warnings make sense.
  6. Compare LOOT's result against this guide's order shape.
  7. Use DML to match the order on Android.

Sorting plugins in Droid Mod Loader

  1. Open DML.
  2. Select the Pocket Mojave profile.
  3. Deploy Current State.
  4. Run plugin discovery.
  5. Enable the guide plugins you installed.
  6. Move official DLC to the top, then YUP, then other fixes, then UI, gameplay, and visual plugins.
  7. Export plugin outputs.
  8. Launch the game from GameNative and check that the expected plugins are active.
Important
Plugin order controls ESM/ESP loading. Mod priority controls loose file conflicts. They are not the same list.

Where the files go

The usual Windows paths are:

%LOCALAPPDATA%/FalloutNV/plugins.txt
%LOCALAPPDATA%/FalloutNV/loadorder.txt

Inside a Proton or Wine prefix, that usually means:

drive_c/users/<user>/AppData/Local/FalloutNV/

The exact GameNative folder can vary. If the game is ignoring your load order, check this first.
